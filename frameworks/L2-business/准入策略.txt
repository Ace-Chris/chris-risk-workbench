# 准入策略

## 概述
准入策略是信贷风控的第一道关卡，决定哪些客户可以进入审批流程。

## 核心要素

### 1. 硬性准入条件
- 年龄范围：22-60岁
- 身份验证：实名认证通过
- 信用记录：无严重不良（M7+逾期）
- 反欺诈：通过设备指纹、活体检测

### 2. 信用准入条件
- 征信查询次数：近3个月 < 6次
- 负债率（DTI）：< 70%
- 在贷机构数：< 3家
- 历史逾期：无M3+记录

### 3. 行为准入条件
- App使用时长 > 30天
- 信息完整度 > 80%
- 近30天活跃天数 > 5天
- 无异常行为（频繁换设备等）

### 4. 业务准入条件
- 行业白名单（非限制行业）
- 地域白名单（非高风险地区）
- 渠道准入（合规获客渠道）

## 准入流程
```
客户申请 → 反欺诈筛查 → 信用准入检查 → 行为准入检查 → 业务准入检查 → 进入审批
                                                              ↓
                                                          拒绝（告知原因）
```

## 拒绝策略
- 硬拒绝：明确不符合条件，直接拒绝
- 软拒绝：当前条件不足，建议补充资料后重新申请
- 延迟：信息不足，进入人工审核

## 准入率监控
- 准入率 = 通过准入的客户数 / 总申请数
- 分渠道准入率对比
- 准入拒绝原因分布分析
- 准入率趋势监控（突然变化可能是攻击）
