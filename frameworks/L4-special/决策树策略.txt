# 决策树策略

## 概述
决策树策略将风控规则组织成树状结构，每个节点是一个判断条件，叶子节点是决策结果。

## 核心优势
- 直观易懂：业务人员可以直接理解规则
- 可解释性强：每条拒绝/通过都有明确原因
- 灵活调整：单条规则可独立修改
- 执行高效：O(树深度)的判断速度

## 构建方法

### 1. 专家规则法
- 由风控专家根据经验编写规则
- 每条规则：条件 → 决策 → 置信度
- 适合冷启动阶段

### 2. 数据驱动法
- 从历史数据中自动挖掘规则
- 使用决策树算法（CART/C4.5）
- 规则剪枝：去除过拟合规则

### 3. 混合法
- 专家规则作为主干
- 数据驱动规则补充细节
- 互相验证，避免盲区

## 决策树结构示例
```
root
├── 征信查询次数 > 6?
│   ├── YES → 拒绝（多头借贷风险）
│   └── NO → 继续
│       ├── DTD（负债率）> 70%?
│       │   ├── YES → 拒绝（过度负债）
│       │   └── NO → 继续
│       │       ├── 信用评分 > 650?
│       │       │   ├── YES → 通过（标准额度）
│       │       │   └── NO → 人工审核
│       │       └── ...
│       └── ...
└── ...
```

## 规则管理

### 规则生命周期
1. 新建规则 → 回测验证 → A/B测试 → 上线 → 监控 → 调整/下线

### 规则冲突处理
- 优先级：硬规则 > 评分卡 > 软规则
- 冲突检测：同一条件不能同时出现在通过和拒绝规则中
- 覆盖度检查：确保所有客群都有规则覆盖

### 版本管理
- 每次规则变更生成新版本
- 可随时回滚到任意历史版本
- 变更审计日志
