# 跨项目经验库

> 本目录存放跨项目可复用的风控经验。由进化师在各项目实践中沉淀，所有项目共享。

## 目录结构

按经验类型分子目录：

```
experiences/
├── feature-engineering/     ← 特征工程经验
├── strategy-design/         ← 策略设计经验
├── data-analysis/           ← 数据分析经验
├── pitfalls/                ← 踩坑记录
└── best-practices/          ← 最佳实践
```

## 文件命名规范

`YYYY-MM-DD | 简短标题.md`

示例：`2025-04-30 | 收入稳定性特征衍生方案.md`

## 文件格式

```markdown
# 标题

## 来源
- 项目：{项目名}
- 日期：YYYY-MM-DD
- 场景：{具体业务场景}

## 经验内容
{详细描述}

## 适用条件
{什么场景下可以复用}

## 注意事项
{可能不适用的场景或风险}
```

## 使用方式

- 进化师在项目中发现可复用经验时，提示用户沉淀到此处
- 用户确认后，进化师将经验写入对应子目录
- 所有项目启动时自动加载经验摘要到 Agent 上下文
