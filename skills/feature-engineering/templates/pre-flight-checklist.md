# Pre-Flight Checklist（写任何特征前必须完成）

> 每次新建特征工程项目时复制此清单，逐项验证。
> 历史教训来源：MTN MoMo Advance 项目（10轮审查，45个错误）

---

## Gate 1: 数据验证（⚠️ 最关键，MTN项目中72%的错误源于此）

### 1.1 字段存在性
- [ ] 已读取原始数据(df)并导出 `df.columns` 列名集合
- [ ] 已与数据字典逐行对照，确认每个字段的实际列名（注意大小写、下划线、拼写）
- [ ] 已标注哪些字段有1m/3m/6m多窗口版本，哪些只有总量
- [ ] 已标注哪些字段在数据中**完全不存在**（不要假设数据字典100%准确）

### 1.2 空值率统计
- [ ] 已统计每个字段的空值率
- [ ] 已区分"业务缺失"（用户从未使用该服务）和"数据质量缺失"（采集/传输丢失）
- [ ] 对空值率>30%的域，计划在L0生成is_missing标志

### 1.3 异常值检测
- [ ] 已对金额字段检查负值
- [ ] 已对日期字段检查不合理年份（如1923年）
- [ ] 已对计数字段检查极端大值
- [ ] 已确认异常值的处理策略（截断/替换/标记）

---

## Gate 2: 命名规范（一旦确定，不可违反）

### 2.1 前缀规则
- [ ] 原始字段清洗后 → 保持原名（例：____）
- [ ] 衍生/加工特征 → 加前缀（例：____）
- [ ] 前缀已写入规范文档，作为single source of truth

### 2.2 缩写映射表
- [ ] 已建立缩写映射表（如 recv/received, bill/bill_pay, bnum/anum）
- [ ] 选定了唯一正确写法
- [ ] 完成后用grep验证无变体残留

### 2.3 命名长度
- [ ] 特征名最大长度已确定（例：≤50字符）
- [ ] 数据类型后缀已确定（如 _flag, _ratio, _amt, _cnt）

---

## Gate 3: 空值处理策略

### 3.1 按类型填值
- [ ] MoMo金额/次数 → 填___
- [ ] GSM数值 → 填___
- [ ] KYC数值 → 填___
- [ ] 类别型 → 填___
- [ ] 计算结果缺失（分母=0等）→ 填___

### 3.2 缺失标志
- [ ] 已确定哪些域需要is_missing标志
- [ ] is_missing标志在空值填充**前**计算（已标注在L0文档中）

## Gate 4: 截断范围定义

| 截断类型 | 范围 | 适用场景 |
|---------|------|---------|
| RATIO | [0, 100] | |
| RATIO_WIDE | [0, 200] | |
| MOMENTUM | [-10, 10] | |
| SLOPE | [-10, 10] | |
| CHANGE_RATE | [-100, 100] | |
| CONCENTRATION | [0, 1] | |
| PERCENT | [0, 1] | |
| FLAG | [0, 1] | |
| AMOUNT | 无截断 | |
| COUNT | 无截断 | |

- [ ] 以上截断类型已确认适用
- [ ] **金额字段不使用SLOPE/MOMENTUM截断范围**

---

## Gate 5: 跨层引用验证

### 5.1 依赖链确认
- [ ] L0输出字段名已导出为"可用字段清单"
- [ ] L1设计的input_fields全部在L0可用清单中
- [ ] L2设计的input_fields全部在L0+L1清单中
- [ ] L3设计的input_fields全部在L0+L1+L2清单中
- [ ] L4设计的input_fields全部在L0+L1+L2+L3清单中

### 5.2 命名一致性
- [ ] L0新增衍生特征的命名（如mtn_m_xxx_recency_days）与L2引用的名字一致
- [ ] **L0输出字段名 ≠ CSV原始列名时，必须以L0输出名为准**

---

## Gate 6: 文档完整性

- [ ] 每个特征的16列全部填写，无空值
- [ ] 16列中无"同上"、"同前"、"-"
- [ ] pseudo_code中无Markdown管道符转义问题（\|）
- [ ] feature_name在整个项目中无重复
